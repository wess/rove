A nice little migration tool for PostgreSQL.

Rove is a fast CLI for managing your Postgres schema. Organize migrations in timestamped folders with up.sql / down.sql, create and drop databases, run migrations and rollbacks, check status, and dump or load your schema.

Features

Schema changes you can read, run, and reverse.

Everything you need to evolve a Postgres database from the command line — and nothing you don't.

📁

Timestamped migrations

Each change lives in its own dated folder with plain up.sql and down.sql files.

⬆️

Up & down

Run all pending migrations with rove up, or revert the most recent one with rove down.

🗄️

Create & drop

Provision or tear down the database named in your DATABASE_URL in a single command.

📊

Status checks

See applied versus pending migrations, with --exit-code and --quiet for CI.

💾

Dump & load

Snapshot the public schema to a SQL file, or load one back in — handy for fresh setups and backups.

🌍

Cross-platform builds

Native binaries for Linux, macOS (Intel & ARM), and Windows — compiled straight from Bun.

⚡

Built on Bun

Written in TypeScript and compiled with Bun for a fast, dependency-light single-file CLI.

🐳

Docker friendly

Point PG_DOCKER_CONTAINER at your container for schema dumps in Docker-only setups.

CLI commands

One verb per task.

The full command surface — short, predictable, and scriptable.

rove init

Create the top-level migrations/ folder.

rove new <name>

Scaffold a new migration directory with up.sql and down.sql.

rove create / drop

Create or drop the database specified by DATABASE_URL.

rove up / migrate

Create the DB if needed and run all pending up.sql scripts.

rove rollback / down

Revert the most recent migration with its down.sql script.

rove status

Show applied vs pending migrations — supports --exit-code and --quiet.

rove dump [file]

Dump the public schema to a SQL file (default schema.sql).

rove load [file]

Load a SQL schema file into the database (default schema.sql).

Migrations structure

Plain SQL, organized by time.

  • ✦Timestamped folders — every migration is its own dated directory, ordered naturally on disk.
  • ✦up.sql applies the change; down.sql reverts it. No DSL — just the SQL you already know.
  • ✦DATABASE_URL (or POSTGRES_URL) points Rove at your Postgres instance.
  • ✦Status in CI — rove status --exit-code fails the build when migrations are pending.
project-root/ └─ migrations/ ├─ 20250515_add_users_table/ │ ├─ up.sql # applies changes │ └─ down.sql # reverts changes └─ 20250601_add_orders_table/ ├─ up.sql └─ down.sql
Get started

Up and running in a minute.

Requires a PostgreSQL instance. Install with Homebrew or Bun, point DATABASE_URL at your database, and go.

terminal 
# 1. install (Homebrew)
brew install wess/packages/rove

# …or as a local dev dependency
bun add -D rove

# 2. point Rove at your database
export DATABASE_URL="postgres://user:pass@host:5432/db"

# 3. initialize and create your first migration
rove init
rove new add_users_table   # edit the up.sql / down.sql it scaffolds

# 4. run migrations and check status
rove up
rove status
